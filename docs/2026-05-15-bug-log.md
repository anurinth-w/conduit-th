# Bug log — Phase 3 Frontend (2026-05-15)

## BUG-001 — JWT Claims ไม่มี `role` และ `name`

**ไฟล์:** `services/auth/service/auth.go`, `services/auth/repository/user.go`

**อาการ:** JWT token ที่ออกจาก auth-svc มีแค่ `uid` และ `email` ไม่มี `name`, `role`, `company_id`

**สาเหตุ:**
- `Claims` struct ไม่มี field `name`
- `User` struct ใน auth repository ไม่มี field `Name` และ Scan ไม่ได้ map ค่า `name` จาก SQL

**แก้ไข:**
- เพิ่ม `Name string` ใน `Claims` struct
- เพิ่ม `Name string` ใน `User` struct
- เพิ่ม `&u.Name` ใน Scan ของ `FindByEmail` และ `FindByID`
- เพิ่ม `Name: user.Name` ใน `generateAccessToken`

**หมายเหตุ:** ตัดสินใจไม่ใส่ `role` และ `company_id` ใน JWT เพราะ 1 user มีได้หลาย company — ใช้ membership picker แทน

## BUG-002 — `GetMemberships` ไม่มี `company_name`

**ไฟล์:** `services/user/repository/user.go`

**อาการ:** endpoint `/v1/users/:id/memberships` return ข้อมูลโดยไม่มี `CompanyName` ทำให้ frontend แสดง company picker ไม่ได้

**สาเหตุ:** query ไม่ได้ JOIN `companies` table

**แก้ไข:**
- เพิ่ม `CompanyName string` ใน `Membership` struct
- แก้ query ให้ `JOIN companies c ON c.id = m.company_id`
- เพิ่ม `c.name` ใน SELECT และ Scan
- เพิ่ม `encoding/json` import สำหรับ `json.Unmarshal` scope

## BUG-003 — Gateway route conflict `:id` vs `:job_id`

**ไฟล์:** `services/gateway/main.go`

**อาการ:** gateway panic ตอน start

```
panic: ':job_id' in new path '/v1/jobs/:job_id/photos' conflicts
with existing wildcard ':id' in existing prefix '/v1/jobs/:id'
```

**สาเหตุ:** Gin ไม่อนุญาตให้ใช้ wildcard ชื่อต่างกันใน path prefix เดียวกัน

**แก้ไข:** เปลี่ยน `:job_id` ทั้งหมดให้เป็น `:id`

```bash
sed -i 's|/jobs/:job_id/|/jobs/:id/|g' services/gateway/main.go
```

## BUG-004 — Gateway ไม่มี CORS middleware

**ไฟล์:** `services/gateway/middleware/cors.go` (ไฟล์ใหม่)

**อาการ:** frontend ที่ `localhost:3001` ยิง request ไป gateway `localhost:8000` ไม่ได้

```
Access to XMLHttpRequest has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present
```

**แก้ไข:** สร้าง `cors.go` middleware และเพิ่มใน `main.go` ก่อน RateLimit

```go
r.Use(middleware.CORS())
```

## BUG-005 — Gateway JWT Claims field ไม่ตรงกับ auth-svc

**ไฟล์:** `services/gateway/middleware/jwt.go`

**อาการ:** token valid แต่ gateway return 401 เพราะ parse claims ไม่ได้

**สาเหตุ:** `Claims` struct ใน gateway ใช้ `json:"user_id"` และ `json:"company_id"` แต่ auth-svc ออก token ด้วย `json:"uid"` และ `json:"cid"`

**แก้ไข:**
```go
// เปลี่ยนจาก
UserID    string `json:"user_id"`
CompanyID string `json:"company_id"`

// เป็น
UserID    string `json:"uid"`
CompanyID string `json:"cid"`
```

## BUG-006 — Services รันโดยไม่มี JWT_SECRET

**อาการ:** gateway validate token ไม่ผ่านเพราะ `JWT_SECRET` ว่าง ทำให้ sign key ไม่ตรงกับ auth-svc

**สาเหตุ:** รัน `go run .` โดยไม่ได้ `source .env` ก่อน

**แก้ไข:** ต้อง source .env ทุกครั้งก่อนรัน service

```bash
set -a && source .env && set +a
cd services/<name> && go run .
```

## BUG-007 — DATABASE_URL ใช้ password ผิด

**ไฟล์:** `.env`

**อาการ:** auth-svc connect DB ไม่ได้ `FATAL: password authentication failed for user "conduit"`

**สาเหตุ:** `.env.example` มี `postgres://conduit:password@...` แต่ Docker Compose ตั้ง `POSTGRES_PASSWORD=conduit_dev`

**แก้ไข:**
```bash
sed -i 's|postgres://conduit:password@|postgres://conduit:conduit_dev@|' .env
```
